> 這是給 Claude 的專案級工作指引，優先級高於全域設定中的通用規則。

## 專案背景

這是一個台灣行銷人員用的 **2026 行銷年度行事曆** 靜態網站，對標 atmarketing.tw/2026calendar/。
詳細規格請閱讀 `SDD.md`，本文件只說明開發習慣與注意事項。

---

## 技術棧

- **單一 `index.html` 架構**：CSS 與 JS 全部寫在 `index.html` 內，不引入外部 JS 框架
- **Vanilla JS**：使用原生 DOM API，不用 jQuery / React / Vue
- **CSS Custom Properties**：暗/亮色主題透過 `[data-theme]` 切換，顏色全部用變數
- **Google Sheets 後台**：資料透過 Google Sheets 發布為 JSON 讀取（免費、無需 API Key）
  - 管理內容：ALL_EVENTS、CPM_ALERTS、HOLIDAYS
  - SOLAR_TERMS 寫死在程式碼內
  - 失敗時使用程式碼內建的預設資料（Fallback）

---

## 開發規則

### 資料優先
- 所有節日、節氣、假日資料，先整理進 JS 常數陣列（`ALL_EVENTS` / `SOLAR_TERMS` / `HOLIDAYS`）
- 資料陣列放在 `<script>` 頂部，與 UI 邏輯分開
- 新增事件時，確認 `month`、`day`、`cat`、`priority` 四個必要欄位都有填

### CSS 規則
- 所有顏色**必須**使用 CSS Custom Properties，禁止 hardcode hex 值在 rule 裡
- 類別顏色對應（不可更改）：
  - `--color-tw`：台灣節慶
  - `--color-ec`：電商檔期
  - `--color-local`：在地活動
  - `--color-intl`：國際日
  - `--color-life`：生活節氣

### JS 規則
- 渲染函式命名：`render*`（如 `renderCards()`、`renderList()`）
- 篩選邏輯集中在 `filterEvents()` 函式，回傳過濾後的陣列，不直接操作 DOM
- **篩選邏輯：類別 × 產業為 AND（交集）關係**
- 事件監聯器用 `addEventListener`，不用 `onclick` attribute
- localStorage 讀寫統一透過 `saveState()` / `loadState()` 封裝
- Google Sheets 資料讀取統一透過 `fetchSheetData(gid)` 封裝，含錯誤處理與 Fallback

## 禁止事項

- ❌ 不引入 React / Vue / Angular
- ❌ 不用 `document.write()`
- ❌ 不 hardcode 顏色（必須用 CSS 變數）
- ❌ 不把 UI 邏輯混入資料陣列
- ❌ 不創建多餘的 .js / .css 分離檔案（初期單一 index.html）

---

## 開發流程

1. **先更新資料** → 再寫 UI → 最後優化動畫
2. 每次新增功能前，先確認 `SDD.md` 的規格，避免偏離設計
3. 新增大量資料（如補完 ALL_EVENTS）時，先寫前 3 筆確認格式正確，再補完其餘
