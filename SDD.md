# SDD — 2026 行銷年度行事曆網站

> 版本：v2.0 | 日期：2026-04-04 | 作者：seanyeh

---

## 1. 專案概述

### 1.1 目標

建置一個台灣行銷人員專用的 2026 年度行事曆網站，整合國定假日、電商檔期、國際節日、24 節氣等資訊，並提供互動式篩選、視圖切換與行銷建議功能。

### 1.2 參考來源

- 設計參考：`src_images/` 資料夾內的三張截圖
- 功能參考：atmarketing.tw/2026calendar/
- 資料來源：行政院人事行政總處公告（國定假日）、農曆曆算（節氣）
- 官方假日 CSV：`scr_data/115年中華民國政府行政機關辦公日曆表_Google行事曆專用.csv`

### 1.3 技術選型

| 項目 | 技術 |
|------|------|
| 前端框架 | **Vanilla JS + HTML5**（無依賴，單一 HTML 檔案優先） |
| 樣式 | **CSS3 Custom Properties**（支援暗色/亮色主題） |
| 資料層 | Google Sheets 發布為 JSON（ALL_EVENTS / CPM_ALERTS / HOLIDAYS），SOLAR_TERMS 寫死在程式碼 |
| 部署 | GitHub Pages / Netlify（靜態網站） |
| 建構工具 | 無（初期），視需要導入 Vite |

> 選用 Vanilla JS 原因：單一 HTML 可直接預覽、無需 build step、易於維護。

## 2. 功能規格

### 2.1 三大視圖模式

#### 視圖 A — 月份卡片（預設）
- 12 張月份卡片，每卡顯示該月所有事件
- 當月卡片加亮邊框效果
- **篩選後**可見事件超過 4 項自動收合，點擊展開
- 「全部收合 / 全部展開」按鈕：切換所有月份卡片的展開狀態，按鈕文字隨狀態切換
- 卡片含：節氣標籤、CPM 警示橫幅、行銷議題建議標籤

#### 視圖 B — 行銷規劃列表
- 欄位：月份 | 週次 | 日期 | 節日名稱 | 行銷類型 | 適合產業 | 建議渠道
- **週次為全年第幾週（W1~W52）**
- 月份跨列合併（rowspan）
- **全部欄位皆可排序**（點擊欄位標題切換升序/降序）

#### 視圖 C — 國定假日 & 節氣
- 分兩區塊：
  1. 2026 國定假日補假表（表格，欄位見 §3.3）
  2. 24 節氣完整列表（Grid 排版）

### 2.2 篩選系統

#### 類別多選篩選（可複選）
| key | 標籤 | 顏色 |
|-----|------|------|
| `tw` | 🇹🇼 台灣 | `#ff6b6b` |
| `ec` | 🛒 電商 | `#34d399` |
| `local` | 📍 在地 | `#fb923c` |
| `intl` | 🌍 國際 | `#a78bfa` |
| `life` | 📚 生活 | `#60a5fa` |

> 另有「全部」按鈕，選取後顯示所有類別。

#### 產業篩選（單選）

| key | 標籤 |
|-----|------|
| `all` | 全產業 |
| `food` | 🍽 餐飲 |
| `beauty` | 💄 美妝 |
| `3c` | 💻 3C |
| `travel` | ✈️ 旅遊 |
| `fashion` | 👗 時尚 |
| `edu` | 📖 教育 |
| `retail` | 🏪 零售 |

#### 篩選邏輯（重要）

類別篩選與產業篩選為 **AND（交集）** 關係：

```
顯示事件 = 事件.cat ∈ 已選類別集合 AND (事件.industry 包含 已選產業 OR 事件.industry 包含 'all')
```

- 選「🛒 電商」+「餐飲」→ 只顯示「既是電商類別 又適合餐飲」的事件
- 選「全部」類別 +「全產業」→ 顯示所有事件

#### 即時搜尋
- `Cmd+K` / `Ctrl+K` 聚焦
- `Esc` 清除
- 關鍵字高亮顯示

### 2.3 倒數計時橫幅

- 顯示最近 3 個 **`priority === 3`** 的重要檔期倒數天數
- 每日自動計算，超過截止日則移至下一個
- 顏色：🔥 熱檔（紅）| 🟠 溫檔（橙）| 普通（藍）

**固定重要檔期（priority 3）清單：**
農曆新年（春節）、西洋情人節、母親節、618年中慶、端午節、中秋節、雙11購物節、黑色星期五、聖誕節

### 2.4 月份快速導航

- 頁面頂端固定 12 個月份按鈕
- 點擊平滑捲動至對應月份區塊
- 當月按鈕加底線 active 狀態
- **手機版：水平滾動**

### 2.5 主題切換

- 預設：暗色主題
- 切換按鈕儲存至 `localStorage`

### 2.6 狀態持久化（localStorage）

| key | 說明 |
|-----|------|
| `cal_theme` | `dark` / `light` |
| `cal_categories` | 已勾選的類別陣列 |
| `cal_industry` | 已選產業 |
| `cal_view` | 當前視圖 (`card`/`list`/`holiday`) |

## 3. 資料結構

### 3.1 資料來源架構

```
Google Sheets（單一檔案，多分頁）
├── Sheet1: ALL_EVENTS     ← 事件資料（最常編輯）
├── Sheet2: CPM_ALERTS     ← CPM 警示（隨市場調整）
└── Sheet3: HOLIDAYS       ← 國定假日（每年依人事行政局公告更新）

程式碼內建（不透過 Google Sheet）
└── SOLAR_TERMS            ← 24 節氣（天文計算，變動極小）
```

**連接方式：** Google Sheets 發布為 JSON（免費、無需 API Key）
- 透過「檔案 > 發布到網路 > JSON」取得公開網址
- 使用 `?gid=分頁ID` 指定讀取不同分頁
- 快取延遲約 5 分鐘，頁面提供「重新整理」按鈕手動載入最新資料
- **Fallback 機制**：若 Google Sheets 無法連線，使用程式碼內建的預設資料

### 3.2 事件（ALL_EVENTS）

```javascript
const ALL_EVENTS = [
  {
    id: 'tw_0101',        // 唯一 ID
    month: 1,             // 月份 1-12
    day: 1,               // 日
    dateLabel: '1/1',     // 顯示用日期
    weekday: 4,           // 星期幾 0=日 6=六
    weekNum: 1,           // 全年第幾週 W1~W52
    name: '元旦',          // 名稱
    cat: 'tw',            // 類別 tw/ec/local/intl/life
    priority: 2,          // 優先度 1-3（3最高，3=重要檔期用於倒數計時）
    industry: ['all'],    // 適合產業
    leadWeeks: 2,         // 建議提前準備週數
    topics: ['新年新氣象'], // 行銷議題標籤
    channel: ['Meta', 'CRM-EDM'], // 建議渠道
    note: '',             // 備註
  },
  // ...
]
```

## 8. 品質要求

- **無 Build 工具依賴**：Phase 1-3 必須可直接用瀏覽器開啟 `index.html` 預覽
- **無外部 CDN 依賴**：不引用 Google Fonts 以外的外部資源（離線可用）
- **資料完整性**：`ALL_EVENTS` 需涵蓋全年至少 80 筆重要事件
- **效能**：首屏渲染 < 1 秒（無 JS 框架負擔）
- **無障礙**：所有互動元素有 `aria-label`，色彩對比度符合 WCAG AA
- **Fallback**：Google Sheets 無法連線時，網站仍可正常運作（使用內建預設資料）
- **視覺品質**：功能對齊參考網站，視覺風格要比參考更精緻
- **動畫流暢**：所有動畫使用 `transform` / `opacity` 避免觸發 layout reflow
